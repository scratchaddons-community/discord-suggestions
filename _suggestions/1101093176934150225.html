That's a confusing title . Let me explain:
In the editors of other game-creation tools like Unity or Unreal, when you play the game anything which gets moved or changed will always reset to what it was in the editor. For example, if you press play then damage an an enemy in the world, when you stop the game the enemy will no longer be damaged, as changes made to game state while playing the project are not applied to the project itself.
Scratch, however, does not work like this. If you set a variable to 0 in the editor then add a script which says "when space key pressed change my variable by 1" then play the project and hit space, the variable will still be 1 even when you stop the project. To experienced Scratch users this seems natural but it becomes more painful in advanced projects. In my project "Scratch Survival" for example, every time I update the project I have to make sure I am looking at the main menu with no world loaded when I save the project, otherwise it would save my world, and anyone who loaded the project would see it. I could just reset everything on green flag clicked, but then I can't make it so if you stop the project and play it again your stuff is saved.
I suggest an addon which adds another button in the editor which would reset the project's state to what it would be for a new user of the project. Any state changes made by the project while it's being played do not change this underlying 'default' state. When the project is saved, the play mode state changes are thrown away and it is saved with the default state only. I think by default the button added by this addon should replace the normal 'stop' button in the editor to make its intended use clearer, but even with this system the stop button could have some niche use (as it deletes all clones and stops scripts without resetting project state) so it should be configurable.

I was imagining basically the editor itself has two states, 'play mode' and 'edit mode'. In edit mode, any changes to variables, sprite positions, or any code you double clicked would actually change the state of the project. When you clicked the flag the editor would then be in 'play mode' where no changes would get saved (except for non-state related changes like changing the code or creating / deleting a sprite or asset). It stays in play mode until you click the new 'reset' button where it reverts back to the default state and edit mode.
{% endraw %}
